# Лабораторная работа №1 — Система проведения хакатонов

## Тема

Разработать серверное приложение на **FastAPI** для организации и проведения
хакатонов. Система должна поддерживать:

- регистрацию участников;
- формирование команд;
- публикацию задач;
- загрузку и оценку работ.

## Требования к модели данных

- ≥ 5 таблиц;
- связи `one-to-many` и `many-to-many`;
- ассоциативная сущность должна иметь **поле, характеризующее связь**,
  помимо ссылок на связанные таблицы.

## Модель предметной области

В рамках работы реализовано 7 таблиц:

| Сущность | Описание |
|---|---|
| `Location` | Площадка проведения хакатона |
| `Hackathon` | Хакатон (описание, статус, FK на `Location`) |
| `Team` | Команда (FK на `Hackathon`) |
| `Participant` | Участник |
| `Task` | Задача хакатона (FK на `Hackathon`) |
| `TeamParticipantLink` | Ассоциативная сущность `Team ↔ Participant` с полем `role` и `joined_at` |
| `Submission` | Ассоциативная сущность `Team ↔ Task` с полями `repo_url`, `score`, `submitted_at` |
| `User` | Пользователь системы (для авторизации) |

### Связи

| Тип | Сущности |
|---|---|
| one-to-many | `Location` → `Hackathon` |
| one-to-many | `Hackathon` → `Team` |
| one-to-many | `Hackathon` → `Task` |
| many-to-many | `Team` ↔ `Participant` через `TeamParticipantLink` |
| many-to-many | `Team` ↔ `Task` через `Submission` |

Обе ассоциативные сущности содержат дополнительные поля, характеризующие
связь (не только внешние ключи), что удовлетворяет критерию ЛР.
