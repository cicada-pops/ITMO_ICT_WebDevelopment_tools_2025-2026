## Что реализовано

### Задание на 9 баллов

- Таблицы, реализованные с помощью **SQLModel** (на базе SQLAlchemy) с
  использованием **PostgreSQL**.
- API c **CRUD** для всех сущностей; GET-запросы, возвращающие модели с
  вложенными объектами для связей `one-to-many` и `many-to-many`.
- Настроенная система миграций с помощью **Alembic** + передача URL БД
  через `.env`.
- Полная аннотация типов в передаваемых и возвращаемых значениях
  API-методов.
- Оформленная файловая структура: код, отвечающий за разную бизнес-логику
  и предметную область, разнесён по отдельным файлам и папкам
  (`app/models/`, `app/auth/`, `app/routers/`).
- Комментарии к ключевым частям кода (docstrings).

### Задание на 15 баллов (подсистема пользователей)

- Регистрация и логин (`POST /auth/register`, `POST /auth/login`).
- Генерация JWT-токенов (через `PyJWT`).
- **Аутентификация по JWT-токену — реализована вручную**, без использования
  сторонних библиотек-обёрток (`OAuth2PasswordBearer`, `fastapi-users`,
  `fastapi-jwt-auth` и т.п.). См. [раздел про авторизацию](lab_1_auth.md).
- Хэширование паролей (через `bcrypt`).
- API для получения текущего пользователя (`GET /users/me`), списка
  пользователей (`GET /users`), пользователя по id (`GET /users/{id}`) и
  смены пароля (`POST /users/me/change_password`).

> По заданию: «Пожалуйста, реализуйте п.3 вручную, без использования
> сторонних библиотек (хэширование и создание JWT не в счёт)». Именно
> п.3 — «Аутентификация по JWT-токену» — и реализован вручную.

## Запуск

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # отредактировать под себя
alembic upgrade head          # накатить миграции
uvicorn main:app --reload
```

Swagger: <http://127.0.0.1:8000/docs>
