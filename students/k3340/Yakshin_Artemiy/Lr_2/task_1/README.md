# Задача 1. threading vs multiprocessing vs async

Программы вычисляют сумму чисел от 1 до `N`. Исходное задание просит
`N = 10^13`, но на чистом Python такая итерация занимает часы — поэтому
для измеримых замеров взят `N = 10^8` (можно поменять через
переменную окружения `N`).

## Запуск

```bash
python sum_threading.py
python sum_multiprocessing.py
python sum_async.py
# или единым прогоном:
python benchmark.py
```

Параметры (env): `N` — верхняя граница, `WORKERS` — количество подзадач.

## Ожидаемый результат

| Подход          | Используется для CPU-bound? | Почему               |
|-----------------|-----------------------------|----------------------|
| threading       | Нет (есть GIL)              | Замедление возможно  |
| multiprocessing | **Да**                      | Реальный параллелизм |
| asyncio         | Нет (одна нить событий)     | Лишь имитация        |
